# Web Technologies Lab Tasks

This repository contains a collection of laboratory assignments completed as part of the Web Technologies course. Each directory represents a specific lab task, progressing from basic front-end structure to dynamic server-side applications.

---

## Repository Structure

The project is organized into modular folders for easy navigation:

* **Lab1/**: Basic HTML structure and semantics.
* **Lab2/**: Responsive E-commerce landing page.
* **Lab3/**: PHP-based Authentication System.
* **Lab4/**: CRUD Operations with Database Integration.
* **Lab5/**: Student Portal with Dynamic Routing and Video Demo.

---

## Lab Details

### Lab 01: Simple HTML Structure

Focuses on the foundational building blocks of the web. This task involves creating a standard document structure using semantic HTML5 tags.

* Key concepts: Headings, paragraphs, lists, and document metadata.

### Lab 02: E-commerce Landing Page

A visually appealing landing page for an online store developed using HTML and CSS.

* Features: Navigation bars, product grids, and call-to-action buttons.
* Focus: CSS Flexbox/Grid for layout and basic responsive design.

### Lab 03: Authentication System

A functional backend implementation using PHP to handle user access.

* **Sign-up**: User registration and data persistence.
* **Login**: Credentials verification and session initiation.
* **Dashboard**: A protected route accessible only to authenticated users.
* **Logout**: Session destruction and redirection.

### Lab 04: CRUD Operations

Implementation of a Create, Read, Update, and Delete system managed through a .txt file.

* Form handling for data entry.
* Displaying records in a tabular format.
* Logic for deleting and modifying existing entries.

### Lab 05: Student Portal (Dynamic Parameters)

A more advanced portal focusing on data identification through URL parameters and dynamic content fetching.

* Uses GET parameters to pass unique identifiers (e.g., student_id) for editing specific records.
* Implements dynamic editing pages that fetch data based on the ID passed in the URL.
